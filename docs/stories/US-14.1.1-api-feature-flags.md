## User Story

As a developer, I want a stable HTTP API for feature flags with per-profile overrides so that the frontend and experiments can toggle behaviour without redeploys.

## Acceptance Criteria

- Endpoints públicos de leitura e endpoints administrativos protegidos (TOTP) quando configurado
- Store persistido em JSON com merge de defaults e overrides por `profile_dir`
- Testes cobrem leitura, escrita autorizada e validação de schema
- Documentação da API referencia o módulo em `backend/feature_flags/`
