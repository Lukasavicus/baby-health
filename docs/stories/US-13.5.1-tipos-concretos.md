## User Story

As a developer, I want concrete types instead of `any` and shared interfaces in dedicated files so that TypeScript catches errors at compile time.

## Acceptance Criteria

- LogSheet.onLog typed with discriminated union
- UIBootstrapPayload fields progressively typed
- Shared interfaces (FeedingEntry, SleepEntry, etc.) in types/ directory
- Zero `any` in public interfaces
